# AI Diffusion State — Project Execution Plan

## Objective

Build an empirical paper for the NBER Economics of AI conference on China by measuring whether China has a distinctive AI diffusion system that converts policy, industrial infrastructure, and smart-factory adoption into measurable industrial upgrading.

The paper's core empirical chain is:

`AI pilot zones -> smart-factory adoption -> industrial AI diffusion -> export and productivity upgrading`

The repo should prioritize reproducibility, source traceability, and a narrow set of credible datasets over breadth.

## Submission target

The NBER conference focuses on the creation, use, and regulation of AI in China, including AI diffusion, productivity, trade, government policy, system-level change, national security, and China-rest-of-world comparisons. Completed papers are due June 4, 2026.

## Core claim

China's AI advantage may be less about frontier-model leadership and more about diffusion capacity. This project measures that capacity through official AI pilot zones, smart-factory recognition, industrial AI project descriptions, city-level industrial outcomes, and product-level export outcomes.

## Minimum viable paper

The minimum viable paper has four empirical outputs.

1. A validated AI pilot-zone treatment table.
2. A clean excellence-level smart-factory adoption dataset for 2024 and 2025.
3. A city-year and city-industry-year diffusion panel.
4. Descriptive and regression evidence linking pilot-zone status, smart-factory adoption, and export upgrading.

## Datasets in scope

### Tier 1, mandatory

1. National New Generation AI Innovation and Development Pilot Zones
   - Role: policy treatment.
   - Unit: city-year.
   - Key fields: city, province, pilot_year, announcement_date, specialization, source_url, source_quality.

2. Excellence-level smart-factory lists
   - Role: main adoption measure.
   - Unit: factory/project-year, aggregated to city-year and city-industry-year.
   - Key fields: firm_name, project_name, province, inferred_city, industry_code, ai_scenario_tags, list_year, source_url, confidence flags.

3. BACI HS6 trade data
   - Role: export outcome.
   - Unit: exporter-importer-HS6-year, collapsed to China-HS6-year or China-HS-sector-year.
   - Key fields: year, hs6, exporter, importer, value, quantity, unit_value, export_growth, unit_value_growth, market_share.

4. EPS or NBS city statistics
   - Role: city controls and industrial outcomes.
   - Unit: city-year.
   - Key fields: GDP, GDP per capita, industrial output, secondary value added, employment, wages, FDI, fixed-asset investment, education, telecom/internet, foreign trade.

### Tier 2, add if time permits

5. AI patent data from CSET/1790, DeepInnovationAI, PATSTAT, or CNIPA.
6. Government procurement data for a narrow sample of Jiangsu, Guangdong, Zhejiang, and Beijing/Shanghai.
7. IFR robotics data or public robot-density proxies.

## Phased plan

### Phase 0 — Repository activation

Deliverables:
- Python package installs locally.
- `make test` runs.
- CI runs on pull requests.
- `/data/raw` is gitignored.
- `/data/processed`, `/outputs/tables`, and `/outputs/figures` are generated by scripts.

Definition of done:
- A new engineer can clone the repo, run setup, run tests, and understand the data workflow from the README.

### Phase 1 — Treatment data

Deliverables:
- `data/processed/pilot_zones.csv`
- `docs/source_notes/pilot_zones.md`

Treatment table requirements:
- Include the first 11 CSET/MOST pilot zones with creation and announcement dates.
- Include the 17-zone December 2021 list.
- Distinguish exact dates from inferred years.
- Include Deqing County as a special county-level unit with mapping notes.
- Include source URL and source type for every row.

Definition of done:
- 17 unique treatment rows.
- No duplicate city names after normalization.
- All treatment years validated or flagged.
- Tests verify row count, required columns, and known pilot-year values for Beijing, Shanghai, Shenzhen, Chengdu, Chongqing, Jinan, and Xi'an.

### Phase 2 — Smart-factory adoption data

Deliverables:
- `data/raw/smart_factories/` with raw HTML/PDF/XLSX files preserved.
- `data/interim/smart_factories_2024_raw.csv`
- `data/interim/smart_factories_2025_raw.csv`
- `data/processed/smart_factories_clean.csv`
- `data/processed/smart_factory_city_year.csv`
- `data/processed/smart_factory_city_industry_year.csv`

Parsing requirements:
- Preserve original Chinese firm and project names.
- Add English translations only as optional fields.
- Infer city only when evidence exists in the project name, company name, province list, company registry, official local notice, or validated manual override table.
- Every inferred city must carry `city_confidence` equal to `exact`, `high`, `medium`, `low`, or `unknown`.
- Every industry assignment must carry `industry_confidence`.
- Every AI scenario tag must be rule-generated from `configs/keywords_zh.yml` or manually overridden with an audit trail.

Definition of done:
- 2024 list reconciles to the official first-batch count.
- 2025 list reconciles to the official 2025 count if used.
- `smart_factories_clean.csv` has no rows without firm_name, project_name, province, list_year, and source_url.
- Aggregations reconcile exactly to the clean project-level table.

### Phase 3 — Export outcomes

Deliverables:
- `data/interim/baci_china_hs6_year.csv`
- `data/processed/export_outcomes_hs6_year.csv`
- `data/processed/export_outcomes_sector_year.csv`
- `docs/source_notes/baci.md`

Requirements:
- Use BACI HS17 as the default for 2017–2024.
- Compute China total exports by HS6-year.
- Compute export value growth, quantity growth, unit value, and unit-value growth.
- Winsorize extreme unit-value growth only in robustness files, never in the canonical raw-derived file.
- Create an explicit HS-to-sector bridge for smart-factory sectors.

Definition of done:
- No negative values for value or quantity.
- Unit value is missing when quantity is zero or missing.
- Growth rates are documented with denominator rules.
- HS-to-sector mappings include confidence flags.

### Phase 4 — City controls

Deliverables:
- `data/processed/city_controls_year.csv`
- `docs/source_notes/city_controls.md`

Requirements:
- Use EPS if accessible; otherwise use NBS yearbook tables and a smaller city sample.
- Normalize city names to the same canonical dictionary used for pilot zones and smart factories.
- Include city-year controls for GDP, GDP per capita, secondary industry, industrial output if available, population, wages, FDI, fixed-asset investment, education, telecom/internet, and foreign trade.

Definition of done:
- City names merge cleanly with pilot zones and smart factories.
- Missingness table produced by variable and year.
- All currency variables document nominal/real status.

### Phase 5 — Baseline analysis

Deliverables:
- `outputs/tables/table_1_dataset_summary.csv`
- `outputs/tables/table_2_top_smart_factory_cities.csv`
- `outputs/tables/table_3_pilot_zone_regressions.csv`
- `outputs/figures/fig_1_diffusion_stack_map.*`
- `outputs/figures/fig_2_pilot_zone_event_study.*`
- `docs/analysis_memo_v1.md`

Baseline models:

1. Descriptive overlap
   - Compare smart-factory counts in AI pilot-zone cities versus non-pilot cities.
   - Report raw differences and controlled differences.

2. City-year adoption model
   - Outcome: count or log(1 + count) of smart-factory projects.
   - Treatment: post-pilot indicator.
   - Controls: city and year fixed effects, or city and province-year fixed effects depending on sample.

3. City-industry adoption model
   - Outcome: smart-factory projects by city-industry-year.
   - Treatment: post-pilot × industry AI exposure.
   - Fixed effects: city-industry, year, and possibly province-year.

4. Export-upgrading model
   - Outcome: export value growth or unit-value growth by HS sector.
   - Exposure: smart-factory sector exposure.
   - Fixed effects: sector and year.

Definition of done:
- Every table is reproducible from scripts.
- Every claim in the memo points to a table, figure, or data source.
- The memo clearly separates descriptive patterns from causal claims.

### Phase 6 — Paper integration

Deliverables:
- `paper/outline.md`
- `paper/data_appendix.md`
- `paper/results_memo.md`
- `paper/red_team_memo.md`

Requirements:
- The paper must avoid overclaiming causality.
- The main contribution should be framed as measurement plus disciplined descriptive and quasi-experimental evidence.
- The red-team memo must identify selection into pilot zones, pre-trends, survivorship, smart-factory recognition bias, and city-industry mapping uncertainty.

## Branching and PR workflow

- Main branch must always run.
- Each engineer works on a feature branch.
- Branch naming convention: `area/short-description`, for example `data/pilot-zones`, `data/smart-factory-parser`, `analysis/event-study`.
- Every PR must include a description of changed files, validation commands run, and known limitations.
- No PR should mix raw-data acquisition, parser changes, and econometric analysis unless explicitly necessary.

## Data governance rules

1. Never edit raw files.
2. Never commit large raw data files unless they are small public seed files.
3. Every processed row must preserve source provenance.
4. Every inferred field must carry a confidence flag.
5. Manual corrections must live in correction tables, not inside generated files.
6. Scripts must be idempotent.
7. Random processes must use fixed seeds.
8. All outputs used in the paper must be generated by scripts, not notebooks.

## Priority order

If time is scarce, prioritize in this order:

1. Pilot zones.
2. 2024 smart-factory list.
3. City-year smart-factory adoption table.
4. Descriptive overlap with pilot zones.
5. BACI export outcomes.
6. City controls.
7. Event study.
8. Patent/procurement/robotics validation.

## Success criterion

The project succeeds if it can support this sentence with evidence:

“China's AI diffusion state is observable in the overlap between national AI pilot-zone policy, official smart-factory adoption, industrial AI project characteristics, and export-relevant manufacturing sectors.”
