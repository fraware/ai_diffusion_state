# Reproducibility

## One-command pipeline

```bash
make setup
make seed
make fetch
make build
make panel
make analysis
make paper
make test
```

Optional (large download, not run in default CI):

```bash
make baci
make analysis
make paper
```

## Idempotency

Re-running `make build`, `make panel`, `make analysis`, or `make paper` overwrites outputs in `data/processed/`, `outputs/`, and `paper/artifact_manifest.json` from the same inputs.

## What CI runs

`.github/workflows/ci.yml`: `seed`, `fetch`, `panel`, `analysis`, `pytest`. BACI steps are skipped when raw BACI files are absent.

## Paper artifacts

After `make analysis`, run `make paper` to write:

- `paper/artifact_manifest.json` — file existence and row counts
- `paper/claim_table_map.csv` — claim-to-table mapping

Narrative memos (not regenerated by scripts): `paper/outline.md`, `paper/data_appendix.md`, `paper/results_memo.md`, `paper/red_team_memo.md`.

## Environment

Python package: `pip install -e .[dev]` (see `pyproject.toml`).

Fixed seeds: any stochastic steps must document seed in script; current pipeline is deterministic given fixed inputs.
